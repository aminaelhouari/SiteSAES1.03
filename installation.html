Protocole d'Installation

Documentation de l'installation de la machine virtuelle

VirtualBox Ubuntu Apache2
1

Préparation du poste

Nous avons choisi comme logiciel de virtualisation Oracle VirtualBox et Ubuntu comme distribution

VirtualBox étant gratuit et open source, ce sont les principales raisons qui nous ont amener à faire ce choix, quant à Ubuntu tout simplement car c’est la distribution Linux la plus connue
2

Configuration de la machine virtuel

Choix des ressources

Nous avons configuré une nouvelle machine virtuelle nommée Ubuntu avec les caractéristiques suivantes:

  • Mémoire RAM 4096 Mo (4 Go)
  • Processeurs 4 Cœurs
  • Disque Dur 80 Go
4 Go de RAM et 4 cœurs permettent de faire tourner l'interface graphique sans faire ralentir l'ordinateur hôte
3

Lancement de l'installation

Après démarrage de la VM, il faut sélectionner “Try or Install Ubuntu”

On a bien fait attention de sélectionner Français comme langue avec un clavier à disposition AZERTY

Étant donné que nous avions des consignes très précises, il était important de sélectionner l'installation interactive, car une installation automatisée aurait été contre-productive pour notre apprentissage.
4

5

Installation des Guest Additions

On va lancer un terminal depuis le CD des additions invités (Pour ça il faut cliquer sur Périphériques -> Insérer l'image CD des Additions invité...).
Un nouveau disque apparaît dans Ubuntu (icône de CD dans la barre de gauche), on clique dessus pour ouvrir son contenu.

Dans la fenêtre du CD, on a fait clic droit dans le dossier puis « Ouvrir dans un terminal ».

Voici les commandes exécutées dans le terminal :

admin-gr106-trio3@Ubuntu-VM: sh autorun.sh
# Ça va nous demander de nous authentifier donc on remet le mot de passe.
admin-gr106-trio3@Ubuntu-VM: sudo apt install bzip2 tar open-vm-tools
# On écrit le mot de passe et on attend que ça finisse de charger puis on écrit o
admin-gr106-trio3@Ubuntu-VM: sudo reboot
Un redémarrage (sudo reboot) a été nécessaire pour appliquer les changements.

7

Création du groupe

On a ouvert le terminal depuis le compte administrateur et écrit ces commandes :

admin-gr106-trio3@Ubuntu-VM: sudo groupadd Trio3
# Ça va nous demander de nous authentifier pour valider la création du groupe
admin-gr106-trio3@Ubuntu-VM: sudo usermod -G Trio3 Trio3_1
admin-gr106-trio3@Ubuntu-VM: sudo usermod -G Trio3 Trio3_2
admin-gr106-trio3@Ubuntu-VM: sudo usermod -G Trio3 Trio3_3
admin-gr106-trio3@Ubuntu-VM: sudo getent group Trio3
sudo groupadd a permis de créer le groupe (nommé Trio3)
sudo usermod -G Trio3 permet de rajouter chaque utilisateur au groupe Trio3
sudo getent group Trio3 permet d'afficher les membres du groupe pour vérifier que tout a fonctionné
8

Personnalisation du terminal

On a ouvert le terminal depuis le compte administrateur et écrit "ls -AL"

On peut apercevoir des fichiers cachés tels que .bash_history ou encore .bashrc (c'est bash rc qui va nous intéresser car c'est un script de configuration pour le terminal)

On lance l'édition en écrivant nano ./.bashrc

admin-gr106-trio3@Ubuntu-VM: export PS1="SAE-S1.03 [\u] \w $
On a juste ensuite copié le fichier bashrc et on l'a collé dans chaque compte utilisateur.
9

Installation du serveur web

On a ouvert le terminal depuis le compte administrateur et écrit ces commandes :

SAE-S1.03 [admin-gr106-trio3] ~ $ sudo apt update
#Ça va nous demander de nous authentifier
SAE-S1.03 [admin-gr106-trio3] ~ $ sudo apt install apache2
#Ça va installer les paquets puis écrire o pour lancer l’installation entière

En écrivant localhost sur Firefox cette page c’est ouvert, ce qui signifie bien que le déploiement est un succès

Nous avons ensuite drag n drop notre dossier dans le dossier docuement de notre compte admin puis copier l'ensemble dans le répertoire racine d'Apache à l’adresse /var/www/html/
